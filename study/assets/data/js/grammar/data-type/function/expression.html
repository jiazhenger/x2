# 用法 1、function 关键字可以用来在一个表达式中定义一个函数 2、函数表达式与函数声明的最主要区别是函数名称（function name），在函数表达式中可省略它 3、一个函数表达式可以被用作一个IIFE（即时调用的函数表达式），它一旦定义就运行 # 语法 let function_expression = function name(param1,param2,...) { statements } # 参数说明 function_expression: 函数表达式名称 name: 函数名称。可被省略，此种情况下的函数是匿名函数（anonymous）。 函数名称只是函数体中的一个本地变量 param1, param2, ...: 可选，被传递给函数的一个参数名称。一个函数至多拥有 255 个参数 statements: 可选，构成函数体的语句 # 返回值 默认情况下，函数是返回 undefined 的。想要返回一个其他的值，函数必须通过一个 return 语句指定返回值 # 函数表达式 var bar = function( ){ } // 常规函数表达式 var bar = function( ){ }( ) // 默认执行，伪装函数定义 var bar = (function( ){ })( ) // 默认执行，伪装函数定义 var bar = function fn( ){ } // 命名函数表达式 # 命名函数表达式 1、即函数有名字 2、如果你想在函数体内部引用当前函数，则需要创建一个命名函数表达式。然后函数名称将会（且只会）作为函数体（作用域内）的本地变量。这样也可以避免使用非标准的 arguments.callee 属性 var bar = function fn( ){ return typeof fn // 函数的名字只在当前函数作用域内有效 => function } console.log(fn) // => undefined bar( ) // 执行函数 # 伪装函数定义 函数表达式在实际应用中还是很常见的，在web开发中有个常用的模式是基于对某种特性的测试来伪装函数定义，从而达到性能优化的目的，但由于这种方式都是在同一作用域内，所以基本上一定要用函数表达式 var bar = function( ){ }( ) // 默认执行，伪装函数定义 var bar = (function( ){ })( ) // 默认执行，伪装函数定义 # 函数表达式不能提升 start( ) // 错误 var start = function( ){ } // 后定义 start( ) // 后执行
