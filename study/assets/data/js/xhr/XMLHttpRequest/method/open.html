# 用法 规定请求的类型、URL 以及是否异步处理请求 # post 与 get 优势 1、GET 比 POST 更简单更快 2、以下情况使用 POST 请求 (1) 无法使用缓存文件（更新服务器上的文件或数据库） (2) 向服务器发送大量数据（POST 没有数据量限制） (3) 发送包含未知字符的用户输入时，POST 比 GET 更稳定也更可靠 # 语法 xhr.open(method, url, async) # 参数 method : 必需。 请求的类型：GET(get)、POST(post) url : 必需。api 请求地址 async : 可选。true（异步）、false（同步） # 示例 xhr.open('GET', /api/search, true) xhr.open('get', /api/search, true) xhr.open('POST', /api/search, true) xhr.open('post', /api/search, true)
